# Pixeleando - Gestión de Campañas Digitales

## Descripción del Proyecto

Este proyecto consiste en el desarrollo de un dashboard de gestión de campañas digitales utilizando React. El objetivo es proporcionar una herramienta interactiva y modular para configurar, lanzar y monitorear campañas en plataformas como Google Ads y Meta Business.

## SPRINT 2: Planificación de Optimización y Segmentación

**Título del Sprint:** Sprint 2 – Optimización y Segmentación
**Subtítulo:** Gestión de Audiencias y Contenido
**Duración:** 20 – 24 de octubre 2025 (Estimado)
**Responsable:** Oliver Montelongo

### Objetivos Propuestos del Sprint 2

1.  Integrar la definición de audiencias segmentadas en el dashboard.
2.  Desarrollar componentes para la gestión básica de contenido de anuncios (texto e imágenes placeholder).
3.  Implementar la visualización de métricas básicas de campañas (clics, impresiones) con datos simulados.

### Backlog del Sprint 2 (Items Prioritarios)

*   Diseño y desarrollo de la interfaz para la segmentación de audiencias.
*   Creación de un módulo para añadir/editar títulos y descripciones de anuncios.
*   Componente para subir y previsualizar imágenes placeholder para anuncios.
*   Integración de gráficos sencillos (placeholder) para mostrar el rendimiento de las campañas.
*   Refinamiento de la navegación entre las secciones del dashboard.

### Avances y Ajustes Realizados (Sprint 2 - Fase Conceptual)

**NOTA IMPORTANTE:**
**Debido a limitaciones temporales y la falta de acceso a un entorno de desarrollo React en este momento, los "avances" descritos para el Sprint 2 son de naturaleza conceptual y de planificación. No ha sido posible implementar código funcional o generar capturas de pantalla de estos progresos. Este enfoque teórico permite mantener la coherencia del proyecto y la planificación, en espera de una futura implementación práctica.**

Los ajustes en esta fase se enfocaron en la priorización y simplificación de las características para asegurar una base sólida para futuras implementaciones, definiendo la estructura y el alcance inicial de cada nueva funcionalidad:

*   **Módulo de Audiencias:** Se planificó una interfaz simplificada para la primera iteración, con foco en la definición de criterios básicos de segmentación, posponiendo características avanzadas.
*   **Gestión de Contenido:** El desarrollo se centró en campos de texto para títulos y descripciones de anuncios, y un área para previsualización de imágenes (usando placeholders), sin integrar la funcionalidad de carga de archivos en este Sprint.
*   **Visualización de Métricas:** Se diseñó la interfaz para mostrar métricas básicas (clics, impresiones) con datos simulados y gráficos simples, aplazando la integración con APIs externas.
*   **Estructura de Componentes:** Se planificó la creación de nuevos componentes modulares para cada funcionalidad (Audiencias, Contenido, Métricas) para asegurar la escalabilidad y el mantenimiento del código.

## Cómo Ejecutar el Proyecto (Base del Sprint 1)

Este proyecto utiliza [React](https://reactjs.org/) y [Node.js](https://nodejs.org/). Para ejecutar la versión base del Sprint 1:

1.  Clona este repositorio:
    ```bash
    git clone https://github.com/olivermontelongo/pixeleando-scrum-sprint1.git
    ```
2.  Navega al directorio del proyecto:
    ```bash
    cd pixeleando-scrum-sprint1
    ```
3.  Instala las dependencias:
    ```bash
    npm install
    ```
4.  Inicia la aplicación en modo desarrollo:
    ```bash
    npm start
    ```
    La aplicación se abrirá automáticamente en tu navegador en `http://localhost:3000`.
